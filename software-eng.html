Chapter 7: Software Engineering

Terms Definitions
Software Engineering the intent of focus on the design, development, testing, and maintenance of software application; while using engineering principles.
Integrated development environment(IDE): is an application that can help programmers to write, test, build, and even package certain software code.
*Association for Computing Machinery (ACM)*: is a professional orgainization
Purpose of ACM: The purpose of ACM is to advance computer techology and the field of it, by uniting educators, researchers, and professionals.
Benefits of ACM membership: having a membership with the ACM has a few benefits but really it has to do with networking. Meeting the people who not only will advance you but your career.
Student ACM membership: this membership provides access to resources for learning, networking, and sharing the advanced knowledge a student would know have access to.
*Institute of Electrical and Electronics Engineers(IEEE)* is a non profit organization for the top computer scientist to have a community.
Purpose of IEEE: is to have a community of technical leaders and to be able to pass information between individuals.
Benefits of IEEE membership: the main benefits of this membership is the learning experience, the collaboration, and the networking of the many conferences.
Student IEEE membership: gives you the access to a very big network of different professionals and to have to opportunity to advance yourself.
*Software Life Cycle:Development Phase* this stage is where the actual coding is started
Requirements analysis: this is where the team as the opportunity to come together and find what this software system needs or should have.
Design: this is the stage where the software developers define the technical details of their product.
Implementation: this phase is where the code is translated into a working product
Testing: This is the process where software is tested to ensure that it mets the objective
Modular Implementation: the developers break down the system break down the system into smaller modules for easier development
Unifed Modeling Language(UML): this is a visual language that helps developers create a new system by drawing diagrams and such.
Use Case: the way a user interacts with the system and/or product to achieve the goal.
Use Case diagram: this diagram describes the high-level functions and the scope of the system.
*Data Relationships:* containers that associate related fields in an application, without storing data themselves
one-to-one: this is a link between two tables where each record in one table is associated with only one record
one-to-many: when a single record in one table can be associated with multiple records in another table
many-to-many: occurs when multiple records in one table are associated with multiple records from another
Design Pattern: is a fundamental concept in object-oriented programming that helps solve common problems in software development
Software quality assurance (SQA): which is the process that monitors and ensures the quality and reliability of the software products that are being used.
*Software Testing*: this is where the product is evaluated to ensure that is meets the expectatons.
Glass box testing: this method examines the internal structure of an application
Black box testing: this method check the fuctionality without knowing how it's coded or even designed.
Beta testing: this sends out a test product to a select group of individuals to see how it is works.
Alpha tesitng: this is the first end-to-end testing of the product to make sure that is meets all requirments
*Documentation* this is the information that describes the certain product and how to use it.
User documentation this is what helps the user to understand how to use the product
System documentation: this is the documents that show the inner working of the system
Technical documentation: this is the instructions on how to use, maintain and build on the software product.
Intellectural property Law this protects the rights of the creators/owners of intellectural property.
Software License: this is the terms and conditions for using or distributing the software
Software Disclaimer this is to notify the additional information to the consumer
Best Defenses againist software liability claims (see last sentence in the chapter) showing that you took care in development and testing, outline the limitations, and proving the right way to use the certain software.
