Chapter 5: Programming Algorithms & Software Testing

Phases of computer program development:

  • Phase 1: Understand the problem
  • Phase 2: Design an algorithm to solve the problem
  • Phase 3: Write the program code in appropraite language based on the algorithm
  • Phase 4: Enter the program and save
  • Phase 5: Execute the program
  • Phase 6: Test the program, find errors and debug, then retest
  • Phase 7: Document and evaluate the program

    • Programming Structures:

  • Sequential structure: Top-down execution
  • Decision structures: if/the/else
  • Iterative (Looping):
  • while loop: continue looping as long as something is true
  • repeat/until loop: continue looping until something is true
  • for loop: loop a given number of times
  • Recursive structures:
  • Repeat a set of instructions as a subtask of itself
  • Repetition continues until:
  • degeneration case, which is a termination condition, is met
  • (degrenerative case is also called a "base case")

    • Two classic programming/database problems:

  • Sorting: Descending order/Ascending order
  • Sorting numbers
  • Sorting strings (in accordance with ASCII values)
  • Searching: find specific data that meet a certain criteria
  • In database:
  • Filters: temporary search
  • Queries: search criteria are saved so they don't have to be recreated
  • Two popular search techniques:
  • Sequential search: search from top to bottom or bottom to top, one item at a time
  • Binary Search:
  • only works if data is SORTED first
  • start in the middle, then go to half of that, etc.

    • Two classic program goals:

  • Efficiency: Program takes minimal time and/or resources
  • Correctness: Program runs in expected ways and works for all expected conditions
    • Software testing and verification Software testing is done to detect:
    Kinds of Tests: Architectural flaws
    Acceptance testing: Verifying whether the whole system works as intended Poor design decisions
    Integration testing: Ensuring that software components or functions operate together Invalid or incorrect functionality
    Unit testing: Validating that each software unit runs as expected. A unit is the smallest testable component of an application. Security vulnerabilities
    Functional testing: Checking functions by emulating business scenarios, based on functional requirements. Black-box testing is a common way to verify functions. Scalability issues
    Performance testing: Testing how the software runs under different workloads. Testing best practices:
    Regression testing: Checking whether new features break or degrade functionality. Continous testing: This type of software testing relies on test automation that is integrated witht he deployment process.
    Stress testing: Testing how much strain the system can take before it fails. Stress testing is considered to be a type of non-fuctional testing. Configuration management: Organizations centrally maintain tet assets and track what software builds to test.
    Usability testing: Validating how well a customer can use a system or web application to complete a task. Servic virtualization: simulates the services and systems that are missing or not yet completed, enabling teams to reduce dependencies and test sooner.
    _______________ Defect or "bug tracking": Monitoring defects is important to both testing and development teams for measuring and improving quality. Automated tools allow teams to track defects.
    _______________ Metrics and reporting: reporting and analytics enable team members to share status, goals and test results. Teams quickly see the overall health of the project and monitor it.
